# Design Systems Index

> **58 real-world design systems** extracted from production websites.
> Each system lives at: `.claude/design-md/{Brand}/DESIGN.md`
>
> **How agents use this index:**
> 1. Scan the table below to identify the best-fit system(s)
> 2. Read the full `DESIGN.md` for any candidate system before making decisions
> 3. Cross-reference "Best For" column with the task at hand
> 4. When in doubt, read 2–3 candidates and pick the closest match

## Reading a DESIGN.md

Every `DESIGN.md` is structured identically:
1. **Visual Theme & Atmosphere** — philosophy, key characteristics
2. **Color Palette & Roles** — semantic color mapping
3. **Typography Rules** — font families, hierarchy table, principles
4. **Component Stylings** — buttons, cards, inputs, navigation patterns
5. **Layout Principles** — spacing, grid, whitespace philosophy
6. **Depth & Elevation** — shadow system, layering strategy
7. **Do's and Don'ts** — guardrails and anti-patterns
8. **Responsive Behavior** — breakpoints, touch targets
9. **Agent Prompt Guide** — quick color reference + example prompts

**Always read sections 7 (Do's/Don'ts) and 9 (Agent Prompt Guide) first** — they provide the fastest actionable guidance.
